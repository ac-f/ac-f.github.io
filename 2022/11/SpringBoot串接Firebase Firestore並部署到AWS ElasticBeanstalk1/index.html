SpringBoot串接Firebase Firestore並部署到AWS ElasticBeanstalk(1)-啟用Firebase服務

發表於2022-11-18|更新於2022-11-19|JavaSpring Boot
|字數總計:794|閱讀時長:2分鐘|閱讀量:

前言

之前在幫以前大學的老師做一個線上測驗的Side Project，

因為小弟實在是太窮了，老師也想節省經費，因此就在想怎麼用最省錢的方式來構建一個系統。

過往是從前端慢慢摸到後端，因此對Firebase不算陌生，因此就把歪腦筋動到了Firebase，

所幸Firebase 也有提供Java的SDK，但是網路上的中文教學範例不是太多，因此今天就想說可以來貢獻下

ps. 這個圖是我自己拉的，總覺得把Spring Boot 的Logo加個帽子(Firebase的Logo)好可愛XD

必要條件

首先你必須滿足以下條件才能跟著教學一步一步完成哦！

  1. 一個Google帳號
  2. 一個AWS帳號
  3. 一台可以摳頂並能連上網路的電腦

使用Firebase服務

Firebase這個服務在2014年被Google收購，因此如果您已經有Google帳號就可以直接開始囉！

首先我們先進入Firebase控制台頁面，

若您之前沒有使用過，映入眼簾的就是這個非常乾淨好看的頁面，這時候我們點一下「建立專案」

這邊就隨便取個喜歡的名字吧，Google Analytic 不在本次教學的範疇內，所以這邊就先不打開囉～（你想開也是可以）

接著就會跳轉到構建專案的畫面，稍微等他跑一下，點擊「繼續」

噠啷～ 不得不說他的介面真的做的非常好看

Cloud Firestore & Realtime Database的差異

在Firebase中有提供兩個服務 Cloud Firestore 與 Realtime Database，這兩個都是NoSQL的Database，並且都有提供實時資料更新的功能（只有前端用得到XD）

他們的差異主要在於Realtime Database更快，但卻不支援複雜的查詢 (其實簡單的就不太好查了)，因此如果你要做的服務需要有稍微複雜的查詢，就請選擇Cloud Firestore

本次範例會使用Cloud Firestore來做示範。

更多內容可以參考官方文件

新增Android應用程式

接著我們需要新增Android應用程式

然後輸入應用程式package的名稱，這邊我輸入我常用的com.acf.demo，點擊「註冊應用程式」，接著就一直按「繼續」就行了。

接著我們點擊 「專案總覽」 > 「專案設定」

然後點「服務帳戶」，並且點擊「產生新的私密金鑰」，將他下載下來，並且改名為firebase.json。

這個檔案很重要，之後要透過這個json file才能存取 Firebase 的資源

接下來我們要啟用Firebase的Cloud Firestore服務，展開控制台左邊導覽列的 「建構」 並點擊 「Firestore Database」

點擊「建立資料庫」

點擊 「以測試模式啟動」 並按 「繼續」。

接著選擇地區的部分可以選擇離自己近的區域，這邊筆者選擇目前所在區域「臺灣」，並按下 「啟用」。

結語

這樣就完成Firebase的設定啦～
